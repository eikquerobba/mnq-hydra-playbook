Framework ก่อนเข้า Setup ทั้ง 2 Model
1. Top-Down Analysis (ก่อน 9:30 AM)
D1 → Trend หลัก + Major Liquidity (BSL/SSL ขนาดใหญ่)
H4 → Intermediate Structure + Key PD Array (FVG, OB)
H1 → Session Bias + จุดที่ Sweep น่าจะเกิด
M5 → Entry Timeframe (ดู IFVG / Breaker ก่อตัว)
2. Liquidity Sweep — หัวใจของ Hydra
ทั้ง 2 Model ต้องมี Liquidity Sweep ก่อนเสมอ

SSL (Sell-Side Liquidity) = แนว Low เดิม / EQL / Stops ใต้ราคา → ถูก Sweep = Bullish Setup
BSL (Buy-Side Liquidity) = แนว High เดิม / EQH / Stops เหนือราคา → ถูก Sweep = Bearish Setup
3. ลำดับเหตุการณ์ที่ต้องเห็น
① HTF Bias ชัด (D1/H4)
② มี Liquidity Pool ให้ Sweep
③ Sweep เกิดขึ้นก่อน Macro Window
④ เข้า Macro Window → หา IFVG หรือ Breaker
⑤ MSS ยืนยัน (สำหรับ Inversion) หรือ Retest (สำหรับ Breaker)
⑥ Entry + SL + TP
4. SMT — ใช้ MES ยืนยันเสมอ
เปิด 2 chart: MNQ | MES

Bullish SMT: MNQ Sweep Low ใหม่ แต่ MES ไม่ Sweep = Buy signal
Bearish SMT: MNQ Sweep High ใหม่ แต่ MES ไม่ Sweep = Sell signal

ถ้า SMT ยืนยัน = ความน่าจะเป็นสูงขึ้นมาก
5. กฎเลือก Model
Inversion (IFVG)
→ ใช้เมื่อ: ต้องการ Entry เร็ว, MSS ชัด, เห็น FVG ล้มเหลว
→ เหมาะกับ: ตลาดที่ Move เร็ว, Strong Reversal

Breaker
→ ใช้เมื่อ: Liquidity Sweep ชัด, OB ล้มเหลว, คาด Aggressive Move
→ เหมาะกับ: LRLR setup, ตลาดมี Momentum ชัดเจน

Model 1 — Inversion (IFVG)
FVG ล้มเหลว → กลายเป็น Resistance/Support ใหม่ + ยืนยัน MSS
Logic หลัก:
FVG ปกติ = ราคาดูดเข้า → แต่ถ้าราคา ทะลุผ่าน FVG ไปได้ แสดงว่า FVG นั้น "ล้มเหลว" (IFVG)
เมื่อราคากลับมา Retest IFVG = เจอ Rejection + MSS = Best Reversal Signal
ก่อน Macro Window — เตรียม
1
Confirm HTF Bias
D1/H4 ชี้ทิศชัด — Bullish หรือ Bearish? ถ้าไม่ชัด = skip วันนี้
2
Mark Liquidity Pool (BSL / SSL)
หาแนว High/Low เดิมที่ยังไม่ถูก Sweep บน H1/H4 — นี่คือเป้าหมายที่ราคาจะวิ่งไป Sweep ก่อน
3
Mark FVG บน H1/M15 ที่น่าจะ Fail
FVG ที่อยู่ใกล้ Liquidity Pool คือตัวเลือกที่ดี — ถ้าราคา Sweep Liquidity แล้วทะลุ FVG นั้น = IFVG เกิด
ใน Macro Window (10:20-10:40 / 10:50-11:10)
4
Liquidity Sweep เกิดขึ้นแล้ว
Bullish: SSL ถูก Sweep (Low เดิมถูกทะลุ) → รอ Buy
Bearish: BSL ถูก Sweep (High เดิมถูกทะลุ) → รอ Sell
5
FVG ถูกทะลุ → IFVG ก่อตัว
ราคาทะลุผ่าน FVG ที่ mark ไว้แล้ว = FVG นั้นกลายเป็น IFVG — zone ที่ราคาจะกลับมา Reject
6
รอราคา Retest IFVG
ราคากลับมาสัมผัส IFVG zone — ดูบน M5/M1 ว่ามี Rejection candle หรือ OB เล็กก่อตัวหรือไม่
7
MSS ยืนยันบน M5
เห็น Market Structure Shift ชัดเจน — แท่งที่ทะลุ Swing High/Low ก่อนหน้า
MSS ชัด = IFVG แข็งแกร่งมาก
8
Entry + SL + TP
Entry: ที่ IFVG zone (Limit order) หรือ Market หลัง MSS
SL: เกิน IFVG zone 2–3 ticks (ถ้า Bullish = ใต้ Low ของ IFVG)
TP1: 1:2 RR | TP2: Opposite Liquidity Pool

Model 2 — Breaker Block
OB ล้มเหลวหลัง Liquidity Sweep → กลายเป็น Breaker ที่มีแรง Rejection สูง
Logic หลัก:
Order Block (OB) ปกติ = แนว Support/Resistance → แต่ถ้าราคา Sweep Liquidity แล้วทะลุ OB ไป
OB นั้น "ล้มเหลว" → กลายเป็น Breaker Block ที่ราคาจะกลับมา Retest แล้ว Reject แรงมาก
Bullish Breaker — Step by Step
1
D1/H4 Bias = Bullish
ตลาดอยู่ในโหมด Bullish — DOL อยู่เหนือราคา (BSL ยังไม่ถูก Sweep)
2
Mark OB ขาลง (Bearish OB) บน H1/M15
แท่งขาขึ้นก่อน Impulse ลง = Bearish OB — จุดที่ถ้าราคาทะลุขึ้นได้ = Breaker ก่อตัว
3
SSL ถูก Sweep (Low เดิมถูกทะลุ)
ราคาดิ่งลง Sweep SSL ก่อน Macro Window — นี่คือการ "จ่ายเงิน" ให้ Smart Money ก่อนวิ่งขึ้น
4
ราคาทะลุ Bearish OB ขึ้นมา = Breaker ก่อตัว
OB เดิมที่เป็นแนวต้านถูกทะลุ → กลายเป็น Bullish Breaker (แนวรับใหม่)
5
ใน Macro Window — รอ Retest Breaker
ราคาวิ่งขึ้นแล้วกลับลงมา Retest Bullish Breaker zone — ดู Rejection บน M5
6
Entry ที่ Breaker + SL + TP
Entry: Limit ที่ Breaker zone (50% ของ zone)
SL: ใต้ Low ของ Breaker zone + 2-3 ticks
TP1: 1:2 RR | TP2: BSL (Buy-Side Liquidity) ถัดไป

2
Mark OB ขาขึ้น (Bullish OB) บน H1/M15
แท่งขาลงก่อน Impulse ขึ้น = Bullish OB — จุดที่ถ้าราคาทะลุลงได้ = Bearish Breaker ก่อตัว
3
BSL ถูก Sweep (High เดิมถูกทะลุ)
ราคาพุ่งขึ้น Sweep BSL ก่อน Macro Window — Smart Money ล้าง Stop ของฝั่ง Long
4
ราคาทะลุ Bullish OB ลงมา = Breaker ก่อตัว
OB เดิมที่เป็นแนวรับถูกทะลุ → กลายเป็น Bearish Breaker (แนวต้านใหม่)
5
ใน Macro Window — รอ Retest Breaker
ราคาดิ่งลงแล้วกลับขึ้นมา Retest Bearish Breaker zone — ดู Rejection บน M5
